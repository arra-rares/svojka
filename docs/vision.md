# ARRA Production — Landing Page Vision

## 1. Objective

Get users to send a request (lead) for event services.

Secondary:

* show past events (gallery)
* build basic trust

---

## 2. Rules

* One primary action: send request
* Pricing visible without clicks
* Max 5–7 sections total
* Above-the-fold shows: service, location, CTA
* Images do the selling; text is short
* No hidden flows, no complex navigation

---

## 3. Target User

Primary:

* people organizing weddings

Secondary:

* corporate events
* private parties / stužková

Implication:

* tone: slightly elegant, but not luxury-heavy
* avoid corporate cold look

---

## 4. Value Proposition

“Premium event experience with instant photo/video output and unique effects.”

Concrete points (must appear in UI):

* instant printing
* unlimited captures
* web gallery after event
* full service (staff included)
* extras (360 video, fog, pyrotechnics, guestbook phone)
* low-budget option: simple Instax rental (instant photos, minimal setup)

---

## 5. Conversion Flow (critical)

### Entry → Action

1. User lands on page

2. Sees main offer + visuals

3. Scrolls → understands services

4. Sees pricing (no friction)

5. Clicks CTA: Check availability

6. Fills form:

   * date
   * location
   * event type
   * number of people

7. Immediate response:

   * “We will confirm availability within 24h (9:00–17:00)”

No booking online. Only qualification.

---

## 6. Page Structure

### 1. Hero (above the fold)

Goal: clarity + hook

Content:

* strong visual (event photo/video)
* headline (what + where)
* subheadline (benefit)
* CTA button

---

### 2. Social Proof (fast trust)

* “100+ events”
* 3–5 testimonials
* optionally Instagram

---

### 3. Services Overview

Goal: quick scan

Blocks:

* Fotobox
* Video 360
* Effects (fog, pyrotechnics, phone)

Each:

* image
* 1–2 lines
* key benefit

---

### 4. Pricing

Goal: remove friction

* Fotobox (2h / 3h / 4h)
* Video 360
* extras

Notes:

* fixed prices
* transport rule (+0.35€/km)

Optional:

* highlight best option

---

### 5. How It Works

Goal: reduce uncertainty

1. Send request
2. We confirm availability
3. Setup at event
4. Enjoy + receive gallery

---

### 6. Gallery

Goal: show real results

* grid of events (cover image only)
* click → password screen
* after password → external link (fotoshare)

---

### 7. CTA Section (repeat)

Goal: capture intent

* strong message
* button → form

---

### 8. Contact

* phone
* email
* WhatsApp
* Instagram / Facebook

---

### 9. Footer

* company info (SK + AT)
* legal
* languages

---

## 7. Design Direction

* clean, modern
* strong visuals (photos > text)
* limited color palette
* subtle premium feel

Avoid:

* clutter
* unnecessary animations
* long text blocks

---

## 8. Language

* default: Slovak
* switch: EN / DE
* same structure across all

---

## 9. Content Strategy

* short, direct
* benefit-driven
* no technical jargon

---

## 10. Technical Notes (deferred)

* CMS needed
* gallery = external link system
* multilingual support
* SEO basics

---

## 11. Success Criteria

* user understands offer in <5 seconds
* form submissions
* low bounce rate
* gallery usage

---

## Notes

* Start simple, iterate after first version
* Do not over-engineer initial release
